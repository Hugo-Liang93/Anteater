# MT5 3D 虚拟办公室动画规范（ANIMATION_SPEC）

## 1. 文档目的

本文档用于规范：
- 角色状态动画
- 区域事件演出
- 流程流转动画
- 2D UI 动效与 3D 动效的边界

目标是让动画成为“业务状态表达”，而不是单纯装饰。

---

## 2. 动画设计原则

### 2.1 动画服务于状态表达
每个动画都应该回答一个问题：
**系统现在发生了什么？**

### 2.2 首版优先做轻动画
优先使用：
- 位移
- 旋转
- 缩放
- 发光
- 透明度变化
- 图标切换

而不是一开始就依赖复杂骨骼动画。

### 2.3 高优先级状态更明显
`error`、`blocked`、`disconnected` 等状态的反馈应强于 `idle` 或 `working`。

### 2.4 动画不抢 UI 信息
动画是辅助观察，不应让用户看不清标签、面板和事件。

---

## 3. 角色基础动画规范

### 3.1 idle（待机）
目标：让角色有生命感，但不显得忙碌。

建议表现：
- 轻微呼吸感
- 轻微上下浮动
- 微弱的身体摆动

### 3.2 working（工作中）
目标：表现模块正在积极处理任务。

建议表现：
- 轻微前倾
- 频率稍快的上下浮动
- 头顶齿轮/处理中图标
- 区域内屏幕或底座有轻微脉冲

### 3.3 thinking / judging（思考中 / 决策中）
目标：表现正在进行策略分析或判断。

建议表现：
- 角色短暂停顿
- 轻微左右摆头或前后微动
- 头顶出现思考图标、问号或判断图标
- 信号员可伴随白板/箭头动画

### 3.4 reviewing（审核中）
目标：表现风控或审核流程正在进行。

建议表现：
- 轻微上下动作减弱
- 头顶出现审核/印章图标
- 风控区出现扫描线或审查灯效果

### 3.5 warning（警告）
目标：提醒当前状态存在异常前兆，但尚未致命。

建议表现：
- 黄灯闪烁
- 黄环脉冲
- 角色略微停顿

### 3.6 error（异常）
目标：强烈提示用户某模块出现问题。

建议表现：
- 红灯快速闪烁
- 红环增强
- 轻微抖动
- 角色或区域亮度变化
- 必要时暂停正常 idle 动画

### 3.7 success / executed / approved（成功类状态）
目标：让用户知道任务已完成、订单已执行或审核已通过。

建议表现：
- 绿色短闪
- 一次性向外扩散光圈
- 轻微抬升后恢复

### 3.8 disconnected / reconnecting
- disconnected：红色静态告警、正常工作动画停止、角色表现为失联
- reconnecting：橙色/蓝色脉冲、周期性闪动、比 disconnected 更有尝试恢复的感觉

## 4. 各角色特定动画建议

### 4.1 MT5 接入员
- `working`：控制台脉冲亮起
- `reconnecting`：周期性闪烁 + 轻微急促动作
- `disconnected`：红灯常亮 + 停止活跃感

### 4.2 行情采集员
- `working`：发光数据球持续流入
- `warning`：数据流变慢、变稀疏
- `error`：数据流中断

### 4.3 指标工程师
- `working`：屏幕脉冲
- `thinking`：图表轻微滚动
- `error`：图表闪黄/闪红

### 4.4 信号策略员
- `judging`：思考图标、方向箭头微动
- `signal_ready`：头顶 BUY / SELL / HOLD 图标短暂弹出
- `error`：决策图标转为红色或出现叉号

### 4.5 风控官
- `reviewing`：审核灯或扫描线
- `blocked`：红色拦截光墙/叉号
- `approved`：绿色印章式闪光

### 4.6 执行员
- `submitting`：终端台高亮
- `executed`：绿光确认
- `rejected`：红色反弹效果

---

## 5. 流程动画规范

### 5.1 主链路线条
线条建议保持常驻，但低亮度。

### 5.2 粒子流动
正常：稳定、流畅、速度中等。
高忙碌：粒子密度增加、速度提升。
warning：粒子变少或颜色偏黄。
error：粒子中断或停止。

---

## 6. 事件演出规范

### 6.1 信号生成
- 信号员局部高亮
- 信号图标短暂出现
- 向风控官方向的线条亮起

### 6.2 风控拒绝
- 风控区红色拦截效果
- 与执行员之间的流程线不继续发光

### 6.3 风控通过
- 风控区绿光
- 到执行员的线条短暂增强

### 6.4 订单执行成功
- 执行区绿光确认
- 底部事件流出现成功事件

### 6.5 模块异常
- 对应角色红灯
- 顶部告警增加
- 底部事件流出现异常消息

---

## 7. UI 动效边界

### 7.1 2D UI 动效由 Framer Motion 负责
包括：
- 面板开合
- 事件流入场
- 状态卡片切换
- tooltip 显隐

### 7.2 3D 动效由 R3F / Three.js 负责
包括：
- 角色动态
- 流程粒子
- 区域闪光
- 告警灯
- 角色高亮

---

## 8. 首版动画优先级

### P0 必做
- idle
- working
- warning
- error
- selected 高亮
- 主链路粒子流动

### P1 建议做
- judging
- reviewing
- blocked
- executed

### P2 可后置
- 行走动画
- 更复杂的骨骼动画
- 表情动画
- 镜头联动

---

## 9. 性能与实现建议
- 动画参数配置化：浮动幅度、闪烁频率、粒子速度、颜色主题
- 避免每帧复杂逻辑
- 正式模型后仍保留状态特效，因为状态灯、光圈、图标更清晰

---

## 10. 验收标准
1. 不看详情也能大致判断角色状态
2. error / warning / success 明显可分辨
3. 主链路具有“正在运转”的感觉
4. 动画不会遮挡 UI 或造成视觉疲劳
5. 替换正式模型后，状态系统仍然可复用
