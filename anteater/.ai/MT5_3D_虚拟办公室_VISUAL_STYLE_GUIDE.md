# MT5 3D 虚拟办公室视觉风格规范（VISUAL_STYLE_GUIDE）

## 1. 文档目的

本文档用于统一 MT5 3D 虚拟办公室项目的整体视觉方向，确保以下内容在同一套风格体系下协作：

- 角色形象
- 办公室场景
- UI 面板
- 动画表现
- 灯光与材质
- 图标与状态提示

目标不是单独定义“一个好看角色”，而是定义一整套可持续执行的视觉语言。

---

## 2. 项目整体视觉定位

本项目的整体视觉定位为：

**温暖卡通 3D 数字工作室风**

它不是：
- 冷硬的金融交易大厅
- 高压监控室
- 写实商务办公室
- 赛博朋克霓虹空间

它更接近：
- 温暖的数字工作室
- 有生命感的 AI 协作空间
- 带轻科技感的卡通办公环境
- 能让人看懂系统在工作、又不会有压迫感的可视化场景

---

## 3. 视觉关键词

整套项目应围绕以下关键词展开：

- 温暖
- 卡通
- 亲和
- 柔和
- 治愈
- 轻科技
- 工作室感
- 非写实
- 清晰状态表达
- 系统可视化

如果某个设计元素与这些关键词明显冲突，应优先判定为不适合。

---

## 4. 角色风格规范

## 4.1 角色总体方向
角色采用 **3D 卡通 / 动画电影感** 路线。

目标气质：
- 可爱但不幼稚
- 亲和但不松散
- 有职业感但不商务化过重
- 有辨识度但仍属于同一个角色宇宙

## 4.2 角色体型与比例
建议统一为：
- 头部相对略大
- 身体圆润、体块简化
- 手脚造型简洁
- 不走写实骨骼或肌肉路线
- 角色整体偏“手办感 / 动画角色感”

不同岗位允许有体型差异，但差异应克制，避免角色像来自不同作品。

## 4.3 面部语言
建议统一：
- 五官简洁
- 眼睛偏圆润、清晰
- 鼻子适度夸张但不写实
- 嘴部简化
- 耳朵可略微夸张
- 眉毛清晰，利于表达情绪
- 表情整体偏温和友好

避免：
- 高写实皮肤细节
- 过多皱纹或真实毛孔
- 过强攻击性表情
- 偏恐怖谷的人脸表现

## 4.4 服装风格
服装建议走：
- 日常工作室风
- 简洁休闲商务风
- 舒适、轻松、可长期观看
- 不使用过多复杂层叠服装

建议搭配：
- 简洁衬衫 / 外套 / T 恤
- 基础裤装
- 适量眼镜、耳机、胸牌、工位配件

避免：
- 纯西装化、过于正式
- 盔甲感、赛博装备感过强
- 复杂多层大面积装饰

## 4.5 岗位差异方式
不同岗位角色不建议完全重做，而应基于统一底模做变体。

建议通过以下方式区分岗位：
- 发型
- 眼镜样式
- 外套或上衣颜色
- 配件（耳机、工牌、平板、印章、终端设备等）
- 工位道具
- 头顶状态图标

这样既能保持统一，又能保证识别度。

---

## 5. 场景风格规范

## 5.1 场景总体方向
场景应定义为：

**数字工作室 / AI 交易工坊**

而不是传统办公室复制品。

空间感觉应是：
- 温暖
- 舒适
- 有功能分区
- 能表现系统流程
- 有适度科技感但不过冷

## 5.2 场景关键词
- 木质
- 柔光
- 工作台
- 数据流
- 创意工坊
- 模块化工位
- 半开放工作空间

## 5.3 场景应避免
- 医疗实验室式冷白空间
- 纯黑服务器机房风
- 高反差霓虹赛博风
- 写实金融大厅风
- 过度复杂的大型游戏关卡感

---

## 6. 色彩体系规范

## 6.1 全局主基调
整体主基调建议为：
- 暖橙
- 杏色
- 米白
- 木色
- 低饱和深灰

这套色彩用于保证整体“温暖卡通工作室”的底色。

## 6.2 功能色 / 岗位色
在暖色基底之上，用少量岗位色区分模块：

- MT5 接入员：蓝色系
- 行情采集员：橙色系
- 指标工程师：绿色系
- 信号策略员：紫色系
- 风控官：红色系
- 执行员：金色系
- 巡检员：青色系
- API 接待员：灰蓝色系

这些颜色应用于：
- 工位点缀
- 头顶标签
- 状态底色
- 局部光效
- UI 角色识别色

## 6.3 状态色
用于运行状态而非岗位识别：
- info：蓝
- warning：黄 / 橙
- error：红
- success：绿

原则：
- 岗位色负责“这是谁”
- 状态色负责“现在怎么了”

---

## 7. 灯光风格规范

## 7.1 灯光总体方向
灯光应以 **暖光、柔和、可读性高** 为核心。

整体建议：
- 主环境光偏暖
- 阴影柔和
- 不走冷白顶灯路线
- 不做高反差舞台打光

## 7.2 局部功能灯光
在暖色总基调下，可以在局部加入冷色点缀，用于体现科技感：
- 数据流可用青蓝点缀
- 图表屏可用轻微冷色光
- 异常区域可用红色警示灯
- 审核/执行区域可用状态色做短暂闪光

## 7.3 灯光应避免
- 通篇冷蓝环境光
- 大面积强阴影
- 过亮高曝光
- 过度电影化导致信息看不清

---

## 8. 材质风格规范

## 8.1 材质总体方向
材质应当：
- 柔和
- 干净
- 轻微手办感
- 非高写实
- 非超强反射

## 8.2 角色材质
角色材质建议：
- 皮肤偏柔和漫反射
- 服装材质简洁，不强调真实布料噪点
- 头发可适度卡通化分块
- 眼镜、按钮、配件有少量高光即可

## 8.3 场景材质
场景材质建议：
- 木质桌面、浅暖色地面
- 屏幕与设备带轻科技感
- 道具材质不要风格混乱
- 避免过多金属镜面

---

## 9. UI 风格规范

## 9.1 UI 总体方向
UI 应与场景统一为：
- 柔和
- 圆角
- 半透明 / 卡片式
- 轻科技但不冰冷
- 信息清晰、层次明确

## 9.2 UI 组件建议
适合使用：
- 圆角面板
- 柔和阴影
- 半透明底板
- 简洁状态标签
- 柔和分隔线

## 9.3 UI 与场景关系
遵循一条核心原则：

**3D 负责感知，UI 负责解释。**

也就是说：
- 3D 场景让你看到“谁在工作、哪里异常、链路怎么走”
- UI 负责告诉你“具体任务是什么、指标是多少、为什么出错”

UI 不应抢夺 3D 主场景的注意力，但必须足够清楚。

---

## 10. 动效气质规范

## 10.1 动效总体气质
动效建议采用：
- 温和
- 清晰
- 有节奏
- 低噪声
- 有业务语义

## 10.2 动效定位
动画不是为了炫技，而是为了让状态被感知。

例如：
- 待机：轻微呼吸感
- 工作中：轻微前倾 / 浮动 / 处理图标
- 决策中：停顿 + 思考图标
- 异常：红灯 + 轻微抖动
- 成功：绿色短闪

## 10.3 动效应避免
- 大量无意义循环炫光
- 像游戏战斗特效一样夸张
- 高频闪动影响阅读
- 角色一直大幅运动造成视觉疲劳

---

## 11. 参考风格结论

以当前参考图为基础，项目角色风格可归纳为：

- Pixar / 动画电影感
- 圆润体块
- 大头但不幼态过重
- 亲和、温暖、轻松
- 适合长期观看
- 适合“数字员工”人格化展示

项目场景风格则应在此基础上扩展为：

- 暖色 3D 办公室 / 工作室
- 创意工坊而非传统金融大厅
- 少量科技元素服务于流程表达
- 角色、场景、UI 都属于同一套世界观

---

## 12. 禁止事项

以下方向原则上不采用：

- 高写实人物风格
- 赛博朋克霓虹风混入主场景
- 冷白机房风替代整体环境
- UI 使用另一套硬质商务后台风
- 每个岗位角色来自完全不同美术体系
- 材质、贴图、灯光风格严重混搭
- 用过多复杂效果掩盖信息表达

---

## 13. 执行建议

实际执行时，建议按以下顺序固化风格：

1. 先确定本文件为项目视觉母规范
2. 角色制作、场景搭建、UI 实现都必须参考本文件
3. 后续再补充更细的：
   - 角色设计表
   - 色板表
   - 灯光参考图
   - UI 组件视觉样例
4. 所有新资源接入前，都要问一个问题：
   - 它是否符合“温暖卡通 3D 数字工作室风”？

如果答案不明确，就不应直接接入主项目。

---

## 14. 结论

这个项目的关键不是“单个人物长得可爱”，而是：

**角色、场景、UI、动效、灯光，都属于同一套温暖卡通 3D 数字工作室世界。**

只有这样，最后的虚拟办公室才会看起来完整、统一、可信，并且真正具备产品感。
