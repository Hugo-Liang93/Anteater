# AGENTS.md - Anteater 前端协作规则

本文件面向在 `D:\Anteater` 工作的 AI Agent。

## 1. 回复语言

- 默认使用简体中文回复
- 说明聚焦三件事：做了什么、为什么这样做、如何验证

## 2. 产品目标

Anteater 前端的目标不是单纯渲染后端数据，而是构建交易决策工作台。
所有改动都应围绕三层展开：

- 业务事实层：展示真实市场、账户、信号、风险和系统状态
- 流程调度层：帮助用户定位流程状态、交接关系、阻塞、风险和机会
- AI 决策层：组织证据并给出建议，但不直接执行交易

## 3. 模块分类准则

新增模块前，必须先回答它属于哪一类，不能直接默认做成员工。

### 员工

只有直接参与交易主链路、承担明确交接职责的节点，才能做成员工。

判断条件：

- 有明确输入、处理、输出
- 会和上下游形成持续交接关系
- 状态变化会直接影响交易流程推进
- 用户需要把它当成工位来定位

如果不满足这些条件，就不要拟人化。

### 支撑模块

如果一个模块主要提供证据、约束、背景条件或验证结果，应做成支撑模块。

典型例子：

- `accountant`：账户、保证金、净值、可用资金
- `calendar_reporter`：经济日历、风险窗口、保护期
- `backtester`：回测、参数验证、稳定性结论
- `inspector`：系统健康、连接状态、队列压力

支撑模块的设计要求：

- 可以保留角色 id，但 UI 不必拟人化
- 要通过“证据”挂接到主链路角色
- 不要和主链路员工处于同一叙事层级

### 数据面板

如果内容主要用于查数、看明细、看日志、看列表，就应放入数据面板。

典型例子：

- 日志
- 告警列表
- 日历明细
- 原始数据流
- 明细信号和行情列表

数据面板不是角色，也不是支撑模块；它只是辅助浏览入口。

## 4. 决策顺序

新增能力时按下面顺序判断：

1. 它是否直接推进交易主链路？
如果是，优先考虑员工。

2. 它是否主要提供证据、约束或背景条件？
如果是，优先做支撑模块。

3. 它是否主要用于原始明细浏览？
如果是，放数据面板。

4. 如果以上都不是，不要强行塞进现有角色体系。
优先考虑独立模块，而不是新增“数字员工”。

## 5. 新增模块评估模板

以后新增模块时，必须先补一份最小评估，不允许直接把后端模块映射成前端员工。

推荐模板：

```text
模块名称：
后端职责：
输入：
输出：
是否直接推进交易主链路：是 / 否
上下游交接：
对决策的作用类型：推进流程 / 提供证据 / 提供约束 / 提供背景 / 提供验证 / 提供明细浏览
用户最需要怎么使用它：看工位状态 / 看证据摘要 / 看原始明细 / 看跨流程总结
最终归类：员工 / 支撑模块 / 数据面板 / 独立模块
前端展示位置：左栏流程角色 / 左栏支撑模块 / 辅助数据面板 / 右侧证据区 / AI 决策区 / 独立页面
```

执行规则：

- 没有这份评估，不要新增员工
- 如果模块更像信息源或约束源，优先归类为支撑模块
- 如果模块更像明细浏览入口，归类为数据面板
- 如果模块跨流程工作，优先做独立模块

快速判断口诀：

- 推流程的，才可能是员工
- 给证据的，优先做支撑模块
- 只查数的，放数据面板
- 跨流程总结的，做独立模块

## 6. 当前明确设计结论

- `accountant` 不是独立员工，应作为账户与保证金支撑模块
- `calendar_reporter` 不是独立员工，应作为风险日历支撑模块
- `backtester` 不是独立员工，应作为研究与验证支撑模块
- `risk_officer` 需要显式读取这些支撑模块提供的证据
- 左栏应承担流程导航和支撑关系表达，不应只是重复点员工

## 7. 关键交互语义

- 左栏：看流程、看支撑模块、看待处理事项
- 主场景：看工位和当前焦点角色
- 右侧详情：看流程详情、员工详情、上游支撑证据
- AI 决策区：看跨流程建议、证据、风险和下一步动作

这四层职责不能混淆。

## 8. AI 决策层原则

- 先模块，后人格
- 先建议，后执行
- 先结构化输出，后自由对话

AI 当前应是独立决策模块，不是新员工，不进入主场景工位链路。

## 9. 代码修改原则

- 业务判断优先放 `src/lib/` 或 `src/config/`
- 配置型数据优先放 `src/config/`
- 组件负责展示，避免在 JSX 中散落复杂逻辑
- 保持中文文案统一
- 涉及交互边界调整时，同步更新 `CLAUDE.md` 与本文件

## 10. 重要目录

- `src/components/layout/`：整体骨架、左栏、顶栏
- `src/components/overlay/`：详情面板、AI 决策模块
- `src/components/panels/`：数据面板与辅助浏览入口
- `src/components/studio/`：工作室场景
- `src/config/employees.ts`：员工 / 支撑模块边界
- `src/config/workflows.ts`：流程与角色分组
- `src/lib/workflowPanel.ts`：流程摘要与待处理事项逻辑
- `src/lib/decisionDesk.ts`：AI 决策上下文和建议生成

## 11. 验证要求

前端改动后至少执行：

```bash
npm run lint
npm run build
```

如无法运行，必须明确说明原因与影响面。
