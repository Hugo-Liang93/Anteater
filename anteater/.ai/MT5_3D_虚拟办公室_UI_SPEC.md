# MT5 3D 虚拟办公室界面规范（UI_SPEC）

## 1. 文档目的

本文档用于定义：
- 页面结构
- 关键 UI 组件
- 状态展示规范
- 颜色与层级规则
- 角色标签与详情面板字段
- 交互行为规范

目标是让界面实现保持一致，避免后续开发出现“同一状态多种展示方式”的问题。

---

## 2. 页面总体结构

首版页面由以下区域组成：

1. **顶部总览栏**
2. **左侧侧边栏**
3. **中间 3D 工作室主场景**
4. **右侧角色详情面板**
5. **底部事件流**

---

## 3. 页面布局规范

## 3.1 顶部总览栏
### 目标
展示全局最重要的系统状态。

### 建议字段
- 当前账户
- 当前 symbol
- 环境：实盘 / 测试
- 总健康度
- 当前告警数
- WebSocket 连接状态

### 设计要求
- 高度保持紧凑
- 重要状态优先靠左或居中
- 连接异常必须明显提示

---

## 3.2 左侧侧边栏
### 目标
提供导航和辅助控制。

### 首版建议内容
- 主工作室
- 监控视图（预留）
- 实验室（预留）
- 过滤器（按模块/按告警级别）

### 设计要求
- 不要过宽
- 不抢占主场景空间
- 支持未来扩展

---

## 3.3 中间 3D 工作室
### 目标
作为主要视觉与交互区，显示：
- 办公室区域
- 角色
- 主链路
- 状态效果
- 告警位置

### 设计要求
- 首版使用固定观察视角
- 保证从左到右的流程观感
- 角色头顶信息不宜过多
- 不用大段文字堆在场景里

---

## 3.4 右侧详情面板
### 目标
对选中的角色进行深入解释。

### 打开方式
- 点击角色
- 点击相关事件（可选）

### 空状态
未选中角色时显示：
- 引导语
- 或简短说明：请选择一个角色查看详情

---

## 3.5 底部事件流
### 目标
展示系统最新事件与动作记录。

### 要求
- 默认按时间倒序或顺序滚动
- 新事件进入要明显但不夸张
- 支持 warning / error / info 区分

---

## 4. 角色标签规范

## 4.1 角色头顶标签字段
首版仅显示以下 3 项：
- 角色名
- 当前状态
- 当前任务简述

### 例子
- 交易判断官
- 决策中
- EMA + ATR 判定

## 4.2 不建议显示
- 太长的错误堆栈
- 过多 metrics
- 复杂表格内容
- 多行长段描述

---

## 5. 详情面板字段规范

建议展示顺序如下：

## 5.1 基本信息区
- 角色名
- 模块名
- 所属区域
- 当前状态
- 更新时间

## 5.2 当前任务区
- 当前任务标题
- 当前 symbol
- 当前处理对象
- 当前阶段说明

## 5.3 核心指标区
根据角色不同显示不同 metrics。

### MT5 接入员
- 连接状态
- 账户
- 最近心跳时间

### 行情采集员
- 当前 symbol
- tick 速率
- 最近报价时间

### 指标工程师
- 当前指标集合
- 最近更新耗时
- 最新结果摘要

### 信号策略员
- 当前策略
- 输出：BUY/SELL/HOLD
- 置信度

### 风控官
- 审核结果
- 冷却剩余时间
- 拒绝原因（如有）

### 执行员
- 最新订单状态
- 成交结果
- 执行耗时

## 5.4 最近事件区
展示最近 3~5 条与该角色相关事件。

## 5.5 异常区
如存在：
- 最近错误
- 风险提示
- 当前告警级别

---

## 6. 状态展示规范

建议统一状态词汇，避免界面文案混乱。

## 6.1 基础状态
- `idle` → 待机
- `working` → 工作中
- `walking` → 行进中
- `thinking` → 思考中
- `waiting` → 等待中
- `warning` → 警告
- `error` → 异常
- `success` → 已完成

## 6.2 业务扩展状态
- `disconnected` → 已断开
- `reconnecting` → 重连中
- `reviewing` → 审核中
- `blocked` → 已拦截
- `approved` → 已放行
- `submitting` → 提交中
- `executed` → 已执行
- `rejected` → 已拒绝
- `judging` → 决策中
- `signal_ready` → 信号已生成

---

## 7. 颜色规范

## 7.1 模块角色主色建议
- MT5 接入员：蓝色系
- 行情采集员：橙色系
- 指标工程师：绿色系
- 信号策略员：紫色系
- 风控官：红色系
- 执行员：金色系
- 巡检员：青色系
- API 接待员：灰蓝色系

## 7.2 告警级别颜色
- `info`：蓝/中性
- `warning`：黄/橙
- `error`：红
- `success`：绿

## 7.3 使用原则
- 角色主色用于识别岗位
- 告警色用于表示当前状态严重程度
- 不要让所有组件都用强对比色

---

## 8. 动效规范

## 8.1 2D UI 动效
适用于：
- 详情面板展开/收起
- 事件流新增
- 顶部告警提醒
- tooltip 出现

### 原则
- 速度快
- 过渡平滑
- 不要过度炫技

## 8.2 3D 状态动效
适用于：
- 角色待机轻微呼吸
- warning 闪黄
- error 闪红/轻抖
- success 绿光短闪
- 主链路粒子流动

### 原则
- 状态可辨识
- 不干扰阅读
- 高优先级状态要更明显

---

## 9. 交互规范

## 9.1 hover 角色
显示：
- 名称
- 状态
- 当前任务

## 9.2 click 角色
行为：
- 高亮选中
- 打开右侧详情
- 场景中可增加轻微聚焦效果（可选）

## 9.3 click 事件
可选行为：
- 自动定位到对应角色
- 打开对应详情

## 9.4 click 空白区域
建议：
- 保持已选中状态，或取消选中，二选一即可
- 不要行为不一致

---

## 10. 空状态与异常状态

## 10.1 无角色选中
显示引导态：
- 请选择一个角色查看详情

## 10.2 无事件
显示：
- 当前暂无事件

## 10.3 无数据
显示：
- 数据加载中 / 暂无数据

## 10.4 WebSocket 断开
顶部必须显示明显提醒：
- 实时连接中断
- 正在重连

---

## 11. 文案规范

## 11.1 文案风格
建议采用：
- 简洁
- 明确
- 偏系统化
- 不使用过度营销语言

## 11.2 任务文案
应使用短句，例如：
- 正在接收 XAUUSD Tick
- 正在计算 EMA / ATR
- 正在生成交易信号
- 正在审核 BUY 指令
- 正在提交订单

---

## 12. 首版 UI 验收标准

1. 页面结构清晰
2. 主场景不被 UI 挤压
3. 角色标签易读
4. 详情面板信息层次清楚
5. 事件流可快速浏览
6. warning / error / success 状态一眼可区分
7. 整体风格统一，不同时出现多套视觉语言
